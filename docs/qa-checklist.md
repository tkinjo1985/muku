# Muku — QA チェックリスト

release ビルド (`target/release/muku.exe`) で一通り流すのが基本。dev ビルドは
ホットリロード等のため挙動が違うので最終確認には使わない。

### テスト環境の前提

- 既定モデル: **Qwen3.5 4B** + GPU（Vulkan）
- モデル切替テストは 2B / 4B / 9B すべて個別に実施
- CPU のみモードは別途実施。2B + CPU では応答が空／JSON パース失敗が発生するので、
  CPU モードでは 4B 以上を使うのが前提（挙動確認用の参考テストとして扱う）

---

## 1. 起動・終了・常駐

- [ ] 初回起動（モデル未 DL）→ Splash「準備中」→「DL 中」→「AI 起動中」→ Chat 画面
- [ ] DL 中にアプリ終了 → 再起動で `.gguf.tmp` から続き（レジューム）
- [ ] 2 回目以降起動 → DL スキップ、「AI 起動中」のみ短時間表示
- [ ] ×ボタン → タスクバーから消えトレイに残る
- [ ] トレイメニュー「終了」→ muku.exe と llama-server 両方消える（タスクマネージャで確認）
- [ ] タスクマネージャで muku.exe を強制終了 → llama-server も自動的に消える（JobObject）
- [ ] muku.exe を 2 回起動 → 新規ウィンドウ出ず既存にフォーカス
- [ ] ウィンドウをリサイズ／移動 → 終了 → 再起動で位置・サイズ保持

## 2. Chat / LLM

- [ ] 1 行メッセージ「おはよう」→ JSON 応答がパースされ吹き出し表示
- [ ] タスク追加「明日までに資料作って」→ Tasks タブに出現、due_at が入る
- [ ] 複数同時追加「来週の予定: A会議、Bレビュー、C提出」→ 全件追加される
- [ ] タスク完了「資料作成終わった」→ Done セクションに移動
- [ ] タスク更新「A 会議の優先度上げて」→ priority が high に
- [ ] タスク削除「C 提出なしで」→ リストから消える
- [ ] 会話途中で Tasks タブに切替 → 応答到着後 Chat タブに戻ると反映済み
- [ ] llama-server を個別に kill → Chat 送信 → 赤いエラーバナー出現
- [ ] エラーバナーの「再送信」→ llama-server 再起動していれば成功
- [ ] エラーバナーの「閉じる」→ バナー消える
- [ ] 長いメッセージ（500 文字程度）送信 → エラーなし
- [ ] LLM が JSON フォーマットを若干崩しても（例: task ネスト忘れ）パースが救済される

## 3. タスク管理

- [ ] Active / Done セクションが状態で分かれる
- [ ] 優先度ドット色: 高=赤、中=橙、低=緑
- [ ] 期限・カテゴリが設定されたタスクで表示される
- [ ] Active 0 件 → 「アクティブなタスクはありません」
- [ ] Done 0 件 → Done セクション自体が非表示
- [ ] 長いタイトル → カード内で折り返し／省略される

## 4. 通知

- [ ] 期限前通知: 「5 分後に〜」→ Settings で 5 分前に設定 → 通知来る
- [ ] 期限超過通知 → 期限を過ぎたタスクで通知（6 時間クールダウン）
- [ ] 定期リマインド → 設定した間隔後に「アクティブなタスク N 件」
- [ ] Settings で通知 OFF → 通知来ない
- [ ] 時間帯外（例: 23:00〜 7:00 の範囲外）→ 定期リマインド来ない
- [ ] アクティブタスク 0 件 → 定期リマインド来ない
- [ ] 通知の送信元は dev/standalone では「Windows PowerShell」（既知、インストーラー化で解決予定）

## 5. 設定（Settings タブ）

- [ ] AI モデル切替 (Qwen3.5 2B ↔ 4B ↔ 9B) が動く
- [ ] 切替時、未 DL モデルなら自動 DL が始まる
- [ ] DL 済みモデルへの切替は即座
- [ ] モデル選択カードに「params / ディスク容量 / 推奨 RAM」が表示される
- [ ] 演算デバイス切替 (GPU ↔ CPU) が動く
- [ ] 各通知設定の変更 → 「保存」→ アプリ再起動 → 値が残る
- [ ] 「通知を有効にする」OFF → 期限前／後／定期すべての select が disabled
- [ ] 「間隔」を無効 → 時間帯 select が disabled
- [ ] 「会話履歴をすべて削除」→ 確認 → 削除 → 再起動後 Chat 履歴が空

## 6. グローバルショートカット・トレイ

- [ ] `Ctrl+Shift+Alt+M`（実環境で空いていた最初の組合せ）でトグル
- [ ] 非表示→表示時に入力欄にフォーカス
- [ ] トレイアイコン左クリック → トグル
- [ ] トレイアイコン右クリック → メニュー（表示/非表示・終了）

## 7. 異常系・ロバストネス

- [ ] ネット切断状態で初回起動 → Splash エラー表示 + 再試行ボタン
- [ ] ネット再接続後に再試行 → DL 成功
- [ ] 会話履歴 500 件超 → 起動時に古いものが自動削除
- [ ] ディスク空き不足で DL → エラー表示（要シミュレーション）
- [ ] SQL 的に怪しい入力（例: `'; DROP TABLE tasks; --`）→ DB 破壊されない（パラメータバインド済み）
- [ ] llama-server がモデルロード失敗 → Splash でエラー＋再試行ボタン
- [ ] モデル切替中にもう一度切替 → 適切に処理（最終選択が反映）

- [ ] タブ切替が即座
- [ ] Chat メッセージ一覧のスクロール
- [ ] Tasks リストのスクロール
- [ ] Settings のスクロール
- [ ] ウィンドウ最小サイズ（360x500）でもレイアウト崩れなし
- [ ] 日本語・英語混在のメッセージ・タスクで崩れなし
- [ ] Splash の進捗バーが滑らかに更新（DL 中）

## 9. アイコン

- [ ] システムトレイ

## 10. データ整合性（手動 SQL 確認）

DB は `%LOCALAPPDATA%\com.takumi.muku\muku.db`。

- [ ] tasks に追加した行が SELECT で確認できる
- [ ] complete 後 status='done' になっている
- [ ] delete 後 行が消えている
- [ ] update 後 updated_at が更新されている
- [ ] messages に user/assistant の両ロールが入る
- [ ] 履歴削除後 messages が空（テーブル自体は残る）
